Windows Server Overview

To use your smartphone as a wireless mouse, your computer needs to understand the touch signals coming from the phone. The Phone Pad PC Server is a lightweight background application designed specifically to translate those Wi-Fi network packets into smooth, native Windows API cursor movements.

Ultra-Lightweight Background Process

Unlike other remote control applications that run heavy UI elements and consume excessive RAM, the Phone Pad Server is designed for efficiency. It initializes a silent local server, displays your local IP address for connection, and then minimizes quietly to your Windows System Tray. It consumes virtually zero CPU power, ensuring your PC runs flawlessly while gaming or working.

Secure Local Network Connection

Your privacy and security are our top priorities. The Phone Pad Server does not route your inputs through the internet or any external cloud servers. The connection happens entirely offline via your local router (WLAN) using fast UDP/TCP protocols. This not only ensures maximum privacy but is also the reason the mouse movement feels completely lag-free.

Firewall Setup & Troubleshooting

Because this application creates a local network server to receive signals, Windows will block it by default.

Step 1: The Initial Prompt When you run the PhonePad_Server.exe for the very first time, a Windows Security Alert popup will appear asking for network access.
Step 2: Allow Access Make sure to check the boxes for "Private networks" and click the "Allow access" button. If you click cancel, your phone will not be able to connect to the PC.

If you accidentally denied access, you can fix it by searching for "Allow an app through Windows Firewall" in your start menu and enabling the checkbox next to Phone Pad.
